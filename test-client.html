Что это?

Сервер истинного времени — единый источник времени для всех игроков. Каждый клиент подключается по WebSocket, измеряет разницу между своими часами и серверными, и может в любой момент узнать точное серверное время.

Как работает синхронизация

  • Клиент отправляет запрос sync_req с локальной меткой t1
  • Сервер получает, ставит метку t2, отправляет ответ с t2 и t3 (момент отправки)
  • Клиент получает ответ, фиксирует t4
  • Вычисляется RTT = (t4 - t1) - (t3 - t2) и offset = ((t2 - t1) + (t3 - t4)) / 2
При подключении делается 5 замеров, выбирается тот, у которого минимальный RTT — он самый точный.

Кнопки на демо-странице

  • Подключиться — устанавливает WebSocket-соединение и автоматически запускает первую синхронизацию (5 замеров)
  • Синхронизировать — принудительная пересинхронизация вручную. Полезна для отладки. В игре синхронизация идёт автоматически каждые 10 минут

Что показывают значения

  • Статус — подключён / отключён от сервера
  • Offset — разница между вашими часами и серверными (в мс). Положительное значение = ваши часы отстают
  • RTT — Round Trip Time, время туда-обратно до сервера. Зелёный <50мс, жёлтый <150мс, красный >150мс
  • Серверное время — текущее время на сервере (вычислено через offset)
  • Локальное время — ваши системные часы

Адрес для подключения

wss://time-server-production.up.railway.app
URL постоянный — не меняется. Используйте его в вашей игре.

Минимальный код (копируй в игру)

Этот модуль подключается к серверу, синхронизирует время и даёт функцию getServerTime():

const TimeSync = (() => { const URL = 'wss://time-server-production.up.railway.app'; const RESYNC_MS = 10 * 60 * 1000; // 10 минут let ws = null; let offset = 0; let synced = false; let _onSync = null; let _syncCb = null; function connect(onSync) { _onSync = onSync || null; ws = new WebSocket(URL); ws.onopen = () => _runSync(); ws.onmessage = (e) => { const msg = JSON.parse(e.data); if (msg.type === 'sync_res' && _syncCb) _syncCb(msg, Date.now()); }; ws.onclose = () => { synced = false; setTimeout(() => connect(_onSync), 3000); }; } function _sendProbe() { return new Promise((resolve) => { const t1 = Date.now(); _syncCb = (msg, t4) => { _syncCb = null; resolve({ rtt: (t4 - msg.t1) - (msg.t3 - msg.t2), offset: ((msg.t2 - msg.t1) + (msg.t3 - t4)) / 2 }); }; ws.send(JSON.stringify({ type: 'sync_req', t1 })); }); } async function _runSync() { const results = []; for (let i = 0; i < 5; i++) results.push(await _sendProbe()); const best = results.reduce((a, b) => a.rtt < b.rtt ? a : b); offset = best.offset; synced = true; if (_onSync) _onSync({ offset, rtt: best.rtt }); setTimeout(() => _runSync(), RESYNC_MS); } return { connect, getServerTime: () => Date.now() + offset, isSynced: () => synced, getOffset: () => offset }; })();

Протокол (если пишете свой клиент)

Всего одна пара сообщений:

// Клиент → Сервер: { "type": "sync_req", "t1": 1740000000000 } // Сервер → Клиент: { "type": "sync_res", "t1": 1740000000000, "t2": 1740000000005, "t3": 1740000000005 }
t1 — когда клиент отправил запрос
t2 — когда сервер получил запрос
t3 — когда сервер отправил ответ
t4 — когда клиент получил ответ (фиксируется на клиенте)
