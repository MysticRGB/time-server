Что это?

Сервер истинного времени — единый источник времени для всех игроков. Каждый клиент подключается по WebSocket, измеряет разницу между своими часами и серверными, и может в любой момент узнать точное серверное время.

Как работает синхронизация

  • Клиент отправляет запрос sync_req с локальной меткой t1
  • Сервер получает, ставит метку t2, отправляет ответ с t2 и t3 (момент отправки)
  • Клиент получает ответ, фиксирует t4
  • Вычисляется RTT = (t4 - t1) - (t3 - t2) и offset = ((t2 - t1) + (t3 - t4)) / 2
При подключении делается 5 замеров, выбирается тот, у которого минимальный RTT — он самый точный.

Кнопки на демо-странице

  • Подключиться — устанавливает WebSocket-соединение и автоматически запускает первую синхронизацию (5 замеров)
  • Синхронизировать — принудительная пересинхронизация вручную. Полезна для отладки. В игре синхронизация идёт автоматически каждые 10 минут

Что показывают значения

  • Статус — подключён / отключён от сервера
  • Offset — разница между вашими часами и серверными (в мс). Положительное значение = ваши часы отстают
  • RTT — Round Trip Time, время туда-обратно до сервера. Зелёный <50мс, жёлтый <150мс, красный >150мс
  • Серверное время — текущее время на сервере (вычислено через offset)
  • Локальное время — ваши системные часы

Адрес для подключения

wss://time-server-production.up.railway.app
URL постоянный — не меняется. Используйте его в вашей игре.

Файл для вставки в игру

Скачайте файл timesync-client.js из репозитория и подключите к вашему проекту. Файл содержит модуль TimeSync — готовый клиент синхронизации.

Файл самодостаточный, без зависимостей. Работает в любом браузере с поддержкой WebSocket.

API модуля TimeSync

TimeSync.connect(opts) — подключиться к серверу TimeSync.disconnect() — отключиться (отменяет реконнект) TimeSync.resync() — принудительная пересинхронизация TimeSync.getServerTime() — текущее серверное время (мс, unix) TimeSync.isSynced() — true если синхронизация прошла TimeSync.getOffset() — offset в мс (серверное − локальное) TimeSync.getRtt() — RTT последней синхронизации (мс)

Параметры connect()

TimeSync.connect({ url: 'wss://...', // необязательно, по умолчанию // wss://time-server-production.up.railway.app onSync: (info) => {}, // вызывается после каждой синхронизации // info = { offset, rtt } onStatus: (status, detail) => {} // status: 'connected'|'disconnected'|'error' });

Быстрый старт (минимальный пример)

<script src="timesync-client.js"></script> <script> TimeSync.connect({ onSync: (info) => { console.log('Offset:', info.offset, 'ms, RTT:', info.rtt, 'ms'); }, onStatus: (status) => { console.log('Статус:', status); } }); </script>

Типичные сценарии в игре

1. Запуск игры только после синхронизации

TimeSync.connect({ onSync: () => { if (!gameStarted) startGame(); }, onStatus: (s) => { if (s === 'disconnected') showReconnectOverlay(); if (s === 'connected') hideReconnectOverlay(); } });

2. Обратный отсчёт до события

const EVENT_TIME = 1740000000000; // unix ms function updateCountdown() { if (!TimeSync.isSynced()) return; const msLeft = EVENT_TIME - TimeSync.getServerTime(); if (msLeft <= 0) { showLabel('Событие началось!'); } else { const sec = Math.ceil(msLeft / 1000); showLabel(`До события: ${sec} сек`); } } setInterval(updateCountdown, 100);

3. Метки времени для действий игрока

function onPlayerAction(action) { sendToGameServer({ action: action, timestamp: TimeSync.getServerTime() }); } // На game-сервере сравнивайте timestamp от разных // игроков напрямую — они в одной шкале

4. Синхронизированная анимация / спавн

const WAVE_INTERVAL = 30000; // волна каждые 30 сек function getCurrentWave() { const t = TimeSync.getServerTime(); return Math.floor(t / WAVE_INTERVAL); } function msUntilNextWave() { const t = TimeSync.getServerTime(); return WAVE_INTERVAL - (t % WAVE_INTERVAL); }

Интеграция с Construct 3

1. Импортируйте timesync-client.js как файл проекта (Files → Import → timesync-client.js) 2. В index.html добавьте перед закрывающим </head>: <script src="timesync-client.js"></script> 3. В событиях Construct используйте через Browser → Execute JS: TimeSync.connect() TimeSync.getServerTime() TimeSync.isSynced()

Интеграция с Unity (WebGL)

1. Положите timesync-client.js в Assets/Plugins/WebGL/ 2. Создайте .jslib файл (Assets/Plugins/WebGL/TimeSync.jslib): mergeInto(LibraryManager.library, { TimeSyncConnect: function() { TimeSync.connect(); }, TimeSyncGetServerTime: function() { return TimeSync.getServerTime(); }, TimeSyncIsSynced: function() { return TimeSync.isSynced() ? 1 : 0; } }); 3. В C# объявите: [DllImport("__Internal")] private static extern void TimeSyncConnect(); [DllImport("__Internal")] private static extern double TimeSyncGetServerTime(); [DllImport("__Internal")] private static extern int TimeSyncIsSynced();

Протокол (если пишете свой клиент)

Всего одна пара сообщений:

// Клиент → Сервер: { "type": "sync_req", "t1": 1740000000000 } // Сервер → Клиент: { "type": "sync_res", "t1": 1740000000000, "t2": 1740000000005, "t3": 1740000000005 }
t1 — когда клиент отправил запрос
t2 — когда сервер получил запрос
t3 — когда сервер отправил ответ
t4 — когда клиент получил ответ (фиксируется на клиенте)

Что делает модуль автоматически

  • При подключении делает 5 замеров, выбирает с минимальным RTT
  • Пересинхронизируется каждые 10 минут
  • Автоматически переподключается при обрыве (с нарастающей задержкой до 60 сек)
  • Таймауты на каждый замер (5 сек) — не зависает при проблемах с сетью
Не создавайте несколько экземпляров TimeSync. Один модуль на всю игру — подключайте при старте, используйте getServerTime() везде где нужно серверное время.
